```javascript
initDarkMode() {
  const isDark = localStorage.getItem('jobika_dark_mode') === 'true';
  if (isDark) {
    this.injectDarkModeStyles();
    document.body.classList.add('dark-mode');
  }
}

injectDarkModeStyles() {
  if (document.getElementById('darkModeStyles')) return; // Prevent duplicates
  
  const style = document.createElement('style');
  style.id = 'darkModeStyles';
  style.textContent = `
    body.dark-mode { background-color: #111827; color: #f9fafb; }
    body.dark-mode .sidebar { background-color: #1f2937; border-right-color: #374151; }
    body.dark-mode .nav-link { color: #9ca3af; }
    body.dark-mode .nav-link:hover, body.dark-mode .nav-link.active { 
      background-color: #374151; color: white; 
    }
    body.dark-mode .card { background-color: #1f2937; color: #f9fafb; border-color: #374151; }
    body.dark-mode .settings-section { background-color: #1f2937; }
    body.dark-mode h1, h2, h3, h4, h5, h6 { color: #f9fafb; }
    body.dark-mode .form-input, input, select, textarea { 
      background-color: #374151; border-color: #4b5563; color: white; 
    }
    body.dark-mode .text-gray-600, .text-gray-700, p { color: #d1d5db !important; }
    body.dark-mode .main-content { background-color: #111827; }
  `;
  
  document.head.appendChild(style);
}
```

**Execution Flow:**
1. Every page loads `session-manager.js`
2. `SessionManager` constructor runs
3. `init()` called ‚Üí `initDarkMode()` executed
4. Checks `localStorage.jobika_dark_mode`
5. If `true`: Injects dark mode CSS + adds `dark-mode` class to body
6. All pages now respect the dark mode setting automatically

## Key Technical Decisions

### 1. Storage Strategy
- **Dark Mode Preference**: `localStorage` (persistent across sessions)
- **Guest Profile**: `sessionStorage` (clears on tab close)
- **Guest Applications**: `sessionStorage` (clears on tab close)

**Rationale:** Preferences should persist, but guest data should be ephemeral to encourage account creation.

### 2. Dark Mode Architecture
- Centralized in `session-manager.js` (DRY principle)
- Dynamic CSS injection (no separate dark mode stylesheet)
- Body class toggle for easy detection

**Rationale:** Single source of truth, easy to extend, minimal file overhead.

### 3. Guest Data Migration Path
- All guest functions mirror logged-in APIs
- Same UI for both user types
- Easy transition: `migrateGuestData()` already exists in session-manager

**Rationale:** Seamless upgrade path when guest creates account.
